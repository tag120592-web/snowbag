# Снеговые мешки — SaaS для проектирования систем мониторинга снеговой нагрузки

Первая версия MVP на основе:
- технического задания (`Техническое_задание_снеговых_мешков.docx`)
- C4-архитектуры (`likec4/`)
- UI-прототипа TN Design System (`ui_kits/snow/`)

## Стек

| Компонент | Технология |
|-----------|------------|
| Frontend | Vue 3, Vite, TypeScript |
| Backend | Go, chi REST API |
| БД | PostgreSQL 16 + PostGIS |
| Кэш/очередь | Redis 7 |
| Файлы | MinIO (S3-совместимое) |

## Быстрый старт (Docker)

```bash
docker compose up --build -d
```

После сборки:

| Сервис | URL |
|--------|-----|
| **Web-приложение** | http://localhost:3000 |
| **API** | http://localhost:8080/health |
| **MinIO Console** | http://localhost:9001 (snowbag / snowbag123) |
| **PostgreSQL** | localhost:5432 (snowbag / snowbag) |

## Разработка frontend локально

```bash
cd web
npm install
npm run dev
```

Frontend на http://localhost:5173 проксирует API на :8080 (инфраструктура должна быть запущена).

## MVP v0.2 — что реализовано

- Список проектов с данными из PostgreSQL
- Мастер из 5 шагов: загрузка → геометрия → климат → расчёт → результат
- **Расчёт снеговых мешков из геометрии кровли** (углы, подветренные зоны, парапеты) по СП 20.13330.2016
- **Автоматическая расстановка датчиков** с ручной корректировкой (перетаскивание)
- Загрузка подложки (PDF/JPG/PNG) в MinIO
- Климат по городу, редактируемые районы и ориентация; роза ветров на схеме
- Переключение 2D/3D, экспорт **JSON, PDF, Excel**
