# План доработок · Модуль «Теплотехника + Паропроницаемость» (ветка `ttr`)

Источник: ТЗ «Автоматизированный теплотехнический расчёт плоской кровли» (СП 50.13330) + UX-прототип
`ТехноНИКОЛь TN Design System (5)/ui_kits/snow/build/StepsThermal.jsx`.

## Зафиксированные решения
1. **Температура помещения** — авто по категории здания (норматив) + возможность ручной правки. (ТЗ поправить: фактически вводов 3 + 1 авто-правка.)
2. **ПИМ** — сразу реальная интеграция (мастер-система по системам и материалам).
3. **Распознавание кровли (в т.ч. DWG/DXF)** — зона коллеги. Теплотехника **потребляет** готовую цифровую модель (контур, элементы, неоднородности).
4. **Паропроницаемость** (разд. 8 СП 50) — входит в объём этой работы.

## Граница с коллегой (во избежание конфликтов)
- **Мои новые файлы** (теплотехника/паро): `api/internal/analytics/thermal*.go`, `api/internal/analytics/vapor*.go`, ПИМ-клиент, новые Vue-экраны под теплотехнику.
- **Общая цифровая модель** (геометрия, препятствия, неоднородности) — читаю, не меняю формат без согласования.
- **Общие файлы под совместную правку** (только маленькими порциями + предупреждать): роутер/мастер шагов, единый экран результата, реестр расчётов («платформа»), `package.json`, `go.mod`, `docker-compose.yml`.

- [ ] Подтвердить нормативную методику с инженером: формулы Rтр, Rусл, r, приведённого Rпр, требуемой толщины (СП 50.13330.2018), и паропроницания (разд. 8).
- [ ] Уточнить мелочи ТЗ: вывод «требуемой толщины теплоизоляции» на экран (да); локальная библиотека материалов (да); тип «проходки» в неоднородностях (да); SSO/журналирование — MVP или позже.

## Этап 1. Модель данных и фундамент бэкенда (Go)
- [ ] Таблицы: `thermal_inputs`, `thermal_results`, `materials` (локальные + кэш ПИМ), `systems` (кэш ПИМ), `heterogeneities` (если не берём целиком из модели снега). Миграция `005_thermal.sql`.
- [ ] Модель проекта расширить ссылкой на теплотехнический расчёт (один объект → несколько расчётов).
- [ ] Чтение общей цифровой модели кровли (геометрия/препятствия) из существующих структур.

## Этап 2. Интеграция с ПИМ (Go)
- [ ] Клиент ПИМ: перечень систем, состав системы, λ материалов, доп. свойства.
- [ ] Кэш в БД + fallback при недоступности ПИМ.
- [ ] Локальная библиотека материалов общего назначения (железобетон, профлист, ЦПС, металл) + добавление пользовательского материала с λ.

## Этап 3. Теплотехнический расчёт (Go, ядро)
- [ ] Климат-привязка по объекту (СП 131): город, ГСОП (Dd), tнаружн., зона влажности — из модели/справочника.
- [ ] Температура помещения: дефолт по категории + ручная правка.
- [ ] Rусл (по составу слоёв, R = t/λ, + Rsi/Rse).
- [ ] Автоопределение теплотехнических неоднородностей из модели: парапеты, шахты, надстройки, **проходки**, примыкания, воронки, локальные элементы → тип, размеры, площадь влияния, вклад в r.
- [ ] Коэффициент теплотехнической однородности `r`.
- [ ] Rпр = Rусл · r; Rтр (требуемое) по нормам; **требуемая толщина теплоизоляции**; вердикт соответствия + запас %.

## Этап 4. Паропроницаемость (Go, разд. 8 СП 50)
- [ ] Проверка сопротивления паропроницанию ограждающей конструкции; плоскость возможной конденсации; защита от накопления влаги.
- [ ] Результат: соответствует/нет + рекомендации (пароизоляция/состав).

## Этап 5. Фронтенд (Vue) — перенос прототипа в реальное приложение
- [ ] Экран «Теплотехника · состав» (StepThermoParams): категория, влажность, температура (авто+правка), система из ПИМ, авто-климат, редактируемый состав, Rусл.
- [ ] Экран «Неоднородности и r» (StepThermoHetero): список неоднородностей из модели, коэффициент r, подсветка на плане.
- [ ] Экран «Результат теплотехники»: Rтр/Rпр/Rусл/r, требуемая толщина, вердикт, разрез конструкции / неоднородности на плане.
- [ ] Экран «Паропроницаемость»: вердикт + диаграмма влаги.
- [ ] Все компоненты — на дизайн-системе TN (как в прототипе).
