# recognizer — автораспознавание чертежей кровли

Сервис превращает чертёж (вектор PDF / DXF / растр) в **размеченную геометрию**
для редактора калькулятора, чтобы не обводить контур руками.

## Жёсткие принципы (из ТЗ)
- **Геометрию даёт только детерминированный парсер**, не модель.
- Единицы — **мм**; без восстановленного масштаба расчёт блокируется.
- **Подтверждение человеком** обязательно для снего-критичных элементов и масштаба.
- Лицензии зависимостей: **MIT / BSD / Apache / MPL** только.
  Запрещены: PyMuPDF/fitz (AGPL), LibreDWG (GPL), ODA (платно). DWG в коде не читаем —
  на вход подаётся векторный PDF/DXF (пользователь экспортирует сам).

## Модель (семантика, не геометрия)
- **Qwen3-VL-8B-Instruct (4-бит, MLX)** локально на Mac (Apple Silicon).
  Проба на реальном чертеже: пик памяти **7.2 ГБ**, ~19 ток/с — влезает с запасом на 24 ГБ.
- Роль: классификация («что это»: воронки, парапеты, перепады, надстройки, подписи).
- Старт — few-shot (без обучения). Дообучение LoRA позже на метках из «маховика» правок.
- Запуск: `mlx-vlm`. Ollama не нужна (с Qwen-VL у неё хуже).

## Стек / окружение
- Python 3.12 через `uv`, venv в `recognizer/.venv` (gitignored).
- `requirements.txt` заморожен. Установка: `uv venv --python 3.12 && uv pip install -r requirements.txt`.
- Зависимости геометрии: `pikepdf` (MPL), `shapely` (BSD), `pdfminer.six` (MIT), `pypdfium2` (BSD) — рендер.

## Этапы
- [x] **0. Каркас + проба модели** — venv, mlx-vlm, Qwen3-VL-8B прогнана на чертеже (читает воронки/уклоны/оси).
- [x] **1a. Геометрия (вектор PDF)** — `vectorgeom.py`: парсер content-stream с привязкой к слоям OCG
      и учётом CTM → polygonize → контур кровли. На тесте `25-АР-4.pdf`: Г-образный контур, 6 вершин,
      парапет (двойная линия) и средняя линия-перепад распознаны. Выдаёт JSON-контракт.
- [x] **1b. Масштаб** — `scale.py`: длиннейшая горизонтальная размерная линия (детерминированно) +
      число над её центром читает Qwen-VL. На тесте: «27000» → mm_per_pt=46.25, сверка с вертикалью
      (14500) сходится в 1%. `pipeline.py` собирает геометрию+масштаб → контур в реальных метрах
      (35.4×14.7 м, 430 м²). HIL: масштаб обязателен к подтверждению.
- [ ] **1c. Числа/отметки** — OCR подписей (текст переведён в кривые → прямого текста нет; нужен OCR,
      напр. tesseract). Габариты, отметки высот.
- [x] **2. Семантика (Qwen-8B)** — `semantic.py`. ВАЖНО: grounding модели (рамки по всей
      картинке) на мелком чертеже НЕточен (рамки промахиваются) — отвергнут. Рабочий подход —
      гибрид: геометрия кластеризует текстовые подписи (слой `*text*`), Qwen-VL читает каждую
      по кропу (надёжно), классификатор по ключевым словам → класс элемента. На тесте: найдены
      2× «воронка с подогревом Ø110» с позициями. `pipeline.py` кладёт элементы в контракт,
      снего-критичные → needs_confirmation. Проверено наложением: контур и воронки на местах.
  - [ ] далее: позиция элемента по концу выноски (точнее центра подписи); надстройки/перепады
        как снего-критичные зоны (левый блок — отдельный уровень → снеговой мешок на границе).
- [x] **3. Встройка в редактор** — РАБОТАЕТ end-to-end:
  - [x] 3a. `recognizer/server.py` — FastAPI на хосте (mlx=Metal, только хост, не Docker): `POST /recognize`.
  - [x] 3b. Go `POST /projects/{id}/recognize` (`api/internal/handler/recognize.go`): MinIO → recognizer
        (`host.docker.internal:8090`, env `RECOGNIZER_URL`) → контракт.
  - [x] 3c. Маппинг контракт → `model.GeometryData`: контур вписан в холст (мм Y-вверх → холст Y-вниз),
        реальная площадь в `AreaM2`, элементы → `Obstacle{circle}`. Проверено: воронки на ендове.
  - [x] 3d. Фронт (`WizardView`): на переходе «Загрузка → Геометрия» зовётся `recognizeProject`,
        контур+элементы подставляются в `RoofCanvas`, кнопка «Распознаём чертёж…», заметка о результате.
        Не перезатирает уже нарисованный контур; при сбое — тихий фолбэк на ручную обводку.
  - [ ] 3e. HIL-бейджи «подтверди» на снего-критичном + лог правок оператора (маховик) — следующее.
- [ ] **4. Маховик + дообучение** — правки → датасет → LoRA (арендованная GPU) → новая версия.
- [ ] **5. Расчёт** — каркас снег/тепло уже есть; геометрия теперь приходит авто.

## Слои тестового чертежа (для профиля layer_rules)
Контур кровли — на `My_thin` (двойная линия = парапет). Имена слоёв у каждого автора свои,
поэтому отбираем **по смыслу**: отбрасываем слои с `axis/dim/text/anno`, из остального берём
сплошные грани, рамку листа отсекаем по габариту, тонкие «кольца» двойных линий — по заполненности.

## Запуск
```bash
cd recognizer
.venv/bin/python vectorgeom.py "<путь к чертежу.pdf>"   # → JSON-контракт в stdout
```
