# Handoff: Анализатор обратной связи (Feedback Analyzer)

## Overview
Веб-приложение службы заботы для анализа клиентской обратной связи. Пользователь загружает выгрузку отзывов (CSV/JSON), сопоставляет колонки файла с полями системы, запускает анализ и получает дашборд с метриками, графиками динамики, кластерами тем и таблицей всех записей.

Поток из 4 шагов (степпер вверху каждого экрана): **Загрузка → Маппинг → Анализ → Дашборд**. В этом пакете 3 готовых экрана: Старт (загрузка), Маппинг (сопоставление колонок) и Дашборд (отзывы).

## About the Design Files
Файлы в `screens/` — это **дизайн-референсы, сделанные в HTML**: прототипы, показывающие задуманный внешний вид и поведение, **а не продакшн-код для прямого копирования**. Исходно это статический экспорт Vue-приложения, поверх которого добавлены стили и скрипты интерактива.

Задача: **воссоздать эти макеты в вашем реальном кодстеке** (React / Vue / и т.д.) по его принятым паттернам и компонентам. Проект уже использует **дизайн-систему ТехноНИКОЛь (TN Life UI Kit)** — если она доступна в кодовой базе как библиотека компонентов (`tn-button`, `tn-card`, `tn-select`, `tn-tabs`, `tn-table`, `tn-input`, `tn-icon` и т.д.), используйте её. Токены системы приложены в `tokens/`.

## Fidelity
**High-fidelity (hifi).** Финальные цвета, типографика, отступы, радиусы, тени и интерактив. Воссоздавайте UI пиксель-в-пиксель, опираясь на компоненты TN-кита. Все значения — в разделе Design Tokens ниже и в `tokens/*.css`.

## Screens / Views

Общий каркас всех экранов:
- **Topbar** (высота ~64px, фон белый): слева логотип-марка (красный квадрат 36×36, радиус ~10px, белая иконка `thumb-up-filled` 20px) + заголовок «Анализатор обратной связи» (16px/700) с подзаголовком «служба заботы · мини-приложения» (12px, `--content-secondary`). Справа — кнопка-ссылка «История» с иконкой `clock`.
- **Stepper** (под топбаром): 4 шага «Загрузка / Маппинг / Анализ / Дашборд». Активный шаг — красный кружок-номер (`--red-60`) + тёмный лейбл; пройденный — состояние `done`; будущий — серый (`--content-tertiary`). Кружок номера 24–28px, между шагами разделители-линии.
- **Контейнер контента**: фон `--background-secondary` (`#f3f5f7`), карточки на белом.

### 1. Старт — «Загрузка» (`screens/01-start.html`)
- **Purpose**: выбрать тип данных и загрузить файл с обратной связью.
- **Layout**: вертикально по центру, максимальная ширина контента ~960px. Заголовок «Какие данные хотите загрузить?» (18px/700) → грид из 4 карточек-типов (`grid-template-columns: repeat(4, 1fr)`, gap 16px) → большая дроп-зона.
- **Components**:
  - **Карточка типа** (`.type-card`): белая, радиус `--radius-lg` (12px), padding 16–20px, иконка сверху (24px), заголовок (14–16px/700), описание (13px, `--content-secondary`). Состояние `active` — рамка `--red-40` (`#f28b92`) + лёгкий розовый фон. 4 типа: «Обращения в поддержку» (иконка chat), «Отзывы пользователей» (star), «Опросы (NPS/CSAT)» (grid), «Другое — настроить вручную» (gear). Выбор одиночный.
  - **Дроп-зона** (`.dropzone`): пунктирная рамка `--border-secondary`, радиус `--radius-xl` (16px), по центру иконка загрузки в белом квадрате с тенью, текст «Перетащите CSV или JSON с обратной связью». Под ней — кнопка «Загрузить демо-данные».

### 2. Маппинг — «Сопоставьте колонки» (`screens/02-mapping.html`)
- **Purpose**: сопоставить колонки из файла с полями системы; задать наименование и тип каждого поля.
- **Layout**: карточка «Сопоставьте колонки» (заголовок 18px/700 + поясняющий текст 13px). Внутри — список строк-маппинга (`.map`, flex-column, gap 6px). Ниже — кнопка «+ Добавить поле». Под карточкой — таблица «Предпросмотр первых 10 строк».
- **Строка маппинга** (`.map-row`, CSS grid, 6 колонок): `[статус 26px] [Наименование ~0.92fr] [Тип 148px] [→ 22px] [Колонка файла ~1.12fr] [× 34px]`, align-items center, радиус 10px, padding 5×8px, hover-подсветка `--neutral-10`.
  - **Статус**: зелёная галочка в кружке `--green-10` фон / `--green-55` (`#009c37`) галочка — когда поле заполнено (есть наименование). Иначе серая точка 8px `--neutral-30`.
  - **Наименование**: `tn-input` (текстовое поле, редактируемое).
  - **Тип**: `tn-select` (Текст / Дата / Оценка / Источник / …).
  - **Стрелка** `→`: цвет `--content-tertiary`, показывает направление «поле системы ← колонка файла».
  - **Колонка файла**: `tn-select` (значения вида text / date / rating / source). Под полем — пример значения (12px, `--content-tertiary`), напр. «напр.: rustore».
  - **×**: кнопка удаления строки. **Верхние 2 строки защищены** (× задизейблен) — обязательные поля.
  - Строка-заголовок: лейблы «Наименование / Тип / Колонка файла» (12px/600, `--content-secondary`) с лёгкой подложкой.
- **Кнопки внизу карточки-навигации**: «← Назад» (на Старт) и «Запустить анализ →» (на Дашборд).

### 3. Дашборд — «Отзывы» (`screens/03-dashboard-reviews.html`)
- **Purpose**: визуальная аналитика по загруженным отзывам.
- **Layout**: вертикальный поток секций (`.dash`, flex-column, gap 20px), карточки на белом. Графики идут парами (`.charts2`, `grid-template-columns: 1fr 1fr`, gap 20px; на узких экранах — 1 колонка).
- **Секции**:
  - **«За что хвалят / за что ругают»**: две колонки тегов/тем с количеством.
  - **«Спросить AI»**: текстовое поле (placeholder «Спросите словами: …») + кнопка «Спросить» + чипы-подсказки («За что обычно ставят 5 звёзд?», «Что больше всего бесит?», «Что изменилось после последнего релиза?»). Клик по чипу подставляет вопрос в поле.
  - **«Динамика рейтинга»** (`.trend-card`): линейный график по неделям. Ось Y — звёзды 5★…1★ (слева, 10px, `--content-tertiary`). Линия — `--red-60` (`#e11b11`), толщина 2.5px, скруглённые стыки, `vector-effect: non-scaling-stroke`. Под линией — градиентная заливка от `rgba(225,27,17,.20)` к прозрачному. Точки данных — круглые маркеры 9px (белая заливка, рамка 2.5px `--red-60`, мягкая тень). Ось X — даты по неделям (формат MM-DD, 10px/600). Над графиком — табы периода «7 дней / 30 дней / Всё время» (`tn-tabs`, акцентный вариант) и подпись среднего «средний 3.22 / 5».
  - **«Динамика по темам»**: стэк-бар-чарт по неделям. Каждая колонка `.col` (радиус верхних углов 5px, overflow hidden) состоит из сегментов `.seg` с цветами тем. Легенда (`.legend`) с цветными маркерами и счётчиками.
  - **«Кластеры»**: группировка тем.
  - **«Все записи»** (`tn-table`): таблица отзывов. Колонки: (пусто) / Текст / Дата / Источник / Оценка / Тема / Тип / Тональность. ~50 строк. Высота строки ~64px, padding ячеек `11px 24px 11px 0`, hover-подсветка `--neutral-10`. Контейнер скроллится по вертикали (max-height ~560px). Над таблицей — фильтры (`tn-select`: Тональность и др.) и кнопка **«Экспорт CSV»** (выгружает таблицу в CSV).

## Interactions & Behavior
- **Навигация между шагами**: клик по шагам степпера и по кнопкам «Назад» / «Запустить анализ» / «Начать заново» переключает экраны. (В прототипе это реализовано переходом между файлами / `postMessage` обёртке; в реальном приложении — это роутинг между шагами одного SPA.)
- **Старт**: выбор карточки типа — одиночный (класс `active`). Дроп-зона и «Загрузить демо-данные» ведут к шагу Маппинг.
- **Маппинг**: «+ Добавить поле» добавляет новую пустую строку (статус — серая точка; при вводе наименования → зелёная галочка). «×» удаляет строку (кроме двух верхних).
- **Дашборд**: чипы заполняют поле вопроса; табы периода/фильтров переключают активное состояние; «Экспорт CSV» формирует и скачивает CSV из таблицы.
- **Переходы**: hover/active на кнопках, табах, строках — `transition: ~.12–.15s ease` (background, box-shadow, border-color, transform). `tn-button:active { transform: translateY(1px) }`.
- **Тени карточек**: `0 1px 2px rgba(30,34,40,.04), 0 10px 28px rgba(30,34,40,.06)`, рамка `1px solid rgba(30,34,40,.05)`.
- **Фокус**: focus-ring `0 0 0 3px rgba(225,27,17,.18)`.

## State Management
Необходимое состояние:
- `uploadType` — выбранный тип данных (один из 4).
- `file` / `rows` — загруженные данные (для демо — сид-набор отзывов).
- `mappings[]` — массив строк маппинга `{ name, type, fileColumn }`; первые два — обязательные.
- `currentStep` — активный шаг (0–3) для степпера и роутинга.
- Дашборд: производные агрегаты (средний рейтинг, динамика по неделям, разбивка по темам/тональности, список записей), фильтры таблицы, текст вопроса к AI.

## Assets
- **Иконки**: TN-спрайт (символы `#icon--…`, формат `<use xlink:href>`, монохром, наследует `currentColor`). В прототипе все 326 символов встроены инлайн в `#svg--iconpack`. Использованные: `thumb-up-filled`, `clock`, `star`, `down-m`, chat/grid/gear и др. В реальном кодстеке используйте библиотеку иконок TN (или эквивалент). НЕ Anthropic-брендинг.
- **Шрифт**: Manrope (Google Fonts) как замена Proxima Nova.
- Растровых изображений в макетах нет.
